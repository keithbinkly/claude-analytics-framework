# Shared Reference

This directory holds cross-domain reference material that is useful across multiple analytics workflows.

Use `knowledge/reference/` for:

- tool references
- standards used across domains
- portable operating checklists
- shared conventions that are broader than a single domain folder

Current promoted examples:

- `tools/dbt-mcp-tools-reference.md`
- `tools/redshift-discovery-snippets.md`
- `standards/folder-structure-and-naming.md`
- `standards/controlled-vocabulary.yml`
- `standards/architecture-validation-checklist.md`
- `standards/mandatory-compile-rule.md`

Use `knowledge/domains/` instead when the content is primarily about one domain such as dbt pipelines or Redshift optimization.

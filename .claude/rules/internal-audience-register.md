# Internal Audience Register

When creating content for internal stakeholders (managers, leadership, business teams):

## DO
- Name the partners, programs, and metrics specifically
- State findings directly: "Samsung's decline rate is 43%"
- Use program-specific terminology the audience already knows
- Frame scope: "Partner-Level Decline Analysis"

## DON'T
- Use narrative hooks: "How 5 partners scaled..." (Substack/blog tone)
- Frame obvious baseline assumptions as revelations: "Why one approach won't work"
- Add vanity stat pills or tech stack credits
- Write for an audience that needs the concept explained

## Test
Read the title aloud. Does it sound like a LinkedIn post or a Slack message to your boss?
If LinkedIn → rewrite. If Slack → ship it.

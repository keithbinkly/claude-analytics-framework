# QA / Execution Handoff Template

CAF-owned handoff template for transferring a pipeline from the builder agent to the QA or execution agent.

Use this when implementation is ready for deeper validation and you want the next agent to inherit context instead of rediscovering it.

```markdown
# QA / Execution Handoff: [Pipeline Name]

## Handoff Metadata

| Field | Value |
|-------|-------|
| Pipeline | [pipeline name] |
| Generated By | [builder agent / person] |
| Generated At | [timestamp] |
| Intended Receiver | [QA agent / execution agent] |
| Priority | [high/medium/low] |
| Estimated QA Time | [time estimate] |
| Parallel Work | [what the builder will do next, if anything] |

---

- variance against legacy or certified output
- duplicate behavior and grain correctness
- incremental behavior if relevant
- edge cases such as NULLs, negatives, or late arrivals
- whether any known limitations are acceptable

## Notes

- Pair this with `qa-report.md`, not instead of it.
